## Overview

`MultiphaseStaggeredHierarchyIntegrator` is the top-level time integrator for the two-fluid (network + solvent) Stokes system. It extends `IBAMR::INSHierarchyIntegrator` and manages the full AMR hierarchy lifecycle: variable registration, solver setup, time advancement, regridding, and visualization output.

**Namespace:** `multiphase`

**Header:** `multiphase/MultiphaseStaggeredHierarchyIntegrator.h`

The integrator drives a staggered MAC (marker-and-cell) discretisation. It owns a `MultiphaseStaggeredStokesOperator` applied inside a PETSc Krylov solver, optionally preconditioned by a `FullFACPreconditioner`.

<ParamField path="object_name" type="string" required>

<ParamField path="input_db" type="SAMRAI::tbox::Pointer<SAMRAI::tbox::Database>" required>

</ParamField>

<ParamField path="grid_geometry" type="SAMRAI::tbox::Pointer<SAMRAI::geom::CartesianGridGeometry<NDIM>>" required>

</ParamField>

<ParamField path="register_for_restart" type="bool" default="true">
  Whether to register this object with the SAMRAI restart manager. Set to `false` when restarts are not needed.
</ParamField>

## Boundary conditions

### `registerPhysicalBoundaryConditions`

```cpp
void registerPhysicalBoundaryConditions(
    std::vector<SAMRAI::solv::RobinBcCoefStrategy<NDIM>*> un_bc_coefs,
    std::vector<SAMRAI::solv::RobinBcCoefStrategy<NDIM>*> us_bc_coefs);
```

Registers physical boundary condition objects for both velocity fields. Only periodic and Dirichlet conditions are currently supported.

<Warning>
  This class does not take ownership of the boundary condition objects. You must manage their lifetime and delete them when finished.
</Warning>

<ParamField path="un_bc_coefs" type="std::vector<SAMRAI::solv::RobinBcCoefStrategy<NDIM>*>" required>
  Robin BC coefficients for the network velocity, one per spatial dimension.
</ParamField>

<ParamField path="us_bc_coefs" type="std::vector<SAMRAI::solv::RobinBcCoefStrategy<NDIM>*>" required>
  Robin BC coefficients for the solvent velocity, one per spatial dimension.
</ParamField>

---

### `registerVolumeFractionBoundaryConditions`

```cpp
void registerVolumeFractionBoundaryConditions(
    SAMRAI::solv::RobinBcCoefStrategy<NDIM>* thn_bc_coefs);
```

Optionally registers boundary conditions for the network volume fraction. Only used when the volume fraction is not an advected quantity. These conditions are ignored at periodic boundaries.

If not set, linear extrapolation is used at physical boundaries.

<ParamField path="thn_bc_coefs" type="SAMRAI::solv::RobinBcCoefStrategy<NDIM>*" required>
  Robin BC coefficient object for the volume fraction.
</ParamField>

## Volume fraction

### `setNetworkVolumeFractionFunction`

```cpp
void setNetworkVolumeFractionFunction(
    SAMRAI::tbox::Pointer<IBTK::CartGridFunction> thn_fcn,
    bool use_as_initial_condition = true);
```

Registers a prescribed (non-advected) function for the network volume fraction. If a function is not registered, the volume fraction is advected with the fluid.

<ParamField path="thn_fcn" type="SAMRAI::tbox::Pointer<IBTK::CartGridFunction>" required>
  Analytic function defining **θ_n** at any point in space and time.
</ParamField>

<ParamField path="use_as_initial_condition" type="bool" default="true">
  When `true`, this function also provides the initial condition for **θ_n**. Set to `false` and call `setInitialNetworkVolumeFraction` separately if a different initial field is required.
</ParamField>

---

### `advectNetworkVolumeFraction`

```cpp
void advectNetworkVolumeFraction(
    SAMRAI::tbox::Pointer<IBAMR::AdvDiffHierarchyIntegrator> adv_diff_integrator,
    bool has_meaningful_mid_value = true);
```

Configures the integrator to transport **θ_n** with the specified advection–diffusion integrator. Also registers that integrator with the hierarchy.

<ParamField path="adv_diff_integrator" type="SAMRAI::tbox::Pointer<IBAMR::AdvDiffHierarchyIntegrator>" required>
  Advection–diffusion integrator responsible for evolving the volume fraction.
</ParamField>

<ParamField path="has_meaningful_mid_value" type="bool" default="true">
  Set to `false` when the transport scheme (e.g. semi-Lagrangian) does not produce a meaningful intermediate-time value after one evolution step. This affects how the midpoint volume fraction is evaluated during time stepping.
</ParamField>

## Usage example

The following is taken from `frm/four_roll_mill.cpp` and shows a complete setup sequence.

```cpp
#include "multiphase/MultiphaseStaggeredHierarchyIntegrator.h"
#include <ibamr/AdvDiffSemiImplicitHierarchyIntegrator.h>
#include <ibtk/muParserCartGridFunction.h>

using namespace multiphase;

// 1. Create the integrator
Pointer<MultiphaseStaggeredHierarchyIntegrator> ins_integrator =
    new MultiphaseStaggeredHierarchyIntegrator(
        "FluidSolver",
        app_initializer->getComponentDatabase("INSVCTwoFluidStaggeredHierarchyIntegrator"),
        grid_geometry,
        true /*register_for_restart*/);

// 2. Create the advection–diffusion integrator for the volume fraction
Pointer<AdvDiffSemiImplicitHierarchyIntegrator> adv_diff_integrator =
    new AdvDiffSemiImplicitHierarchyIntegrator(
        "AdvDiffIntegrator",
        app_initializer->getComponentDatabase("AdvDiffIntegrator"),
        true /*register_for_restart*/);

// 3. Set physical parameters
const double eta_n = input_db->getDouble("ETA_N");
const double eta_s = input_db->getDouble("ETA_S");
const double xi    = input_db->getDouble("XI");
const double nu_n  = input_db->getDouble("NU_N");
const double nu_s  = input_db->getDouble("NU_S");
ins_integrator->setViscosityCoefficient(eta_n, eta_s);
ins_integrator->setDragCoefficient(xi, nu_n, nu_s);

// 4. Set initial conditions
Pointer<CartGridFunction> un_init =
    new muParserCartGridFunction("un", app_initializer->getComponentDatabase("un"), grid_geometry);
Pointer<CartGridFunction> us_init =
    new muParserCartGridFunction("us", app_initializer->getComponentDatabase("us"), grid_geometry);
Pointer<CartGridFunction> p_init  =
    new muParserCartGridFunction("p",  app_initializer->getComponentDatabase("p"),  grid_geometry);
ins_integrator->setInitialData(un_init, us_init, p_init);

// 5. Set up and advect the network volume fraction
Pointer<CartGridFunction> thn_init_fcn =
    new muParserCartGridFunction("thn_init", app_initializer->getComponentDatabase("thn"), grid_geometry);
ins_integrator->setInitialNetworkVolumeFraction(thn_init_fcn);
ins_integrator->advectNetworkVolumeFraction(adv_diff_integrator);

// 6. Register scaled body forces
Pointer<CartGridFunction> fn_fcn =
    new muParserCartGridFunction("FN_FCN", app_initializer->getComponentDatabase("FN_FCN"), grid_geometry);
Pointer<CartGridFunction> fs_fcn =
    new muParserCartGridFunction("FS_FCN", app_initializer->getComponentDatabase("FS_FCN"), grid_geometry);
ins_integrator->setForcingFunctionsScaled(fn_fcn, fs_fcn);

// 7. Initialize the patch hierarchy
ins_integrator->initializePatchHierarchy(patch_hierarchy, gridding_algorithm);

// 8. Time step loop
while (!IBTK::rel_equal_eps(loop_time, time_end) && ins_integrator->stepsRemaining())
{
    double dt = ins_integrator->getMaximumTimeStepSize();
    ins_integrator->advanceHierarchy(dt);
    loop_time += dt;
}
```
